Sobre o Estacionar

O Estacionar é uma plataforma completa de gerenciamento de Estacionamento. O sistema oferece suporte a múltiplos clientes em uma arquitetura multi-tenant, garantindo isolamento total de dados entre clientes.

Características Principais

  • Gravação em Nuvem: Armazenamento seguro dos dados.
  • Multi-Tenant: Cada cliente opera de forma independente com seus próprios clientes e dados
  • Gestão de Carros: Cadastro e gerenciamento completo de Carros estacionados
  • Cadastro de serviços adicionais: Ideal para adicionar serviços a mais para os clientes finais.
  • Emissão de cupom: Emissão do cupom fiscal sobre o serviço prestado pelo estacionamento.
  • Calculo do valor total: Veja o valor e o tempo total do veiculo estacionado.
Isolamento de Dados: Como cliente, você terá acesso apenas aos seus próprios dados. Todos os endpoints retornam automaticamente apenas os registros relacionados ao seus dados. Não é possível visualizar ou acessar dados de outros estacionamentos.
Autenticação: A maioria dos endpoints requer autenticação via token JWT obtido no endpoint de login. O token deve ser incluído no header Authorization: Bearer {token} em todas as requisições autenticadas.

Carros

POST /vehicles ▼
Cadastrar um novo carro (Registrar entrada)

Headers

Nome Valor Obrigatório
Accept application/json SIM
Content-Type application/json SIM

Parâmetros do Body (JSON)

Parâmetro Tipo Descrição Obrigatório
license_plate string Placa do veículo (ex: FHH8346) SIM
model string Modelo do veículo (ex: Ford Ka) SIM
locale string Localização da vaga (ex: Subsolo 2, Vaga 12) SIM

Exemplo de Request (cURL)

curl --request POST \
  --url https://api.estacionar.com/vehicles \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --data '{
    "license_plate": "fhh8346",
    "model": "Ford Ka",
    "locale": "Subsolo 2, Vaga 12"
}'

Resposta de Sucesso

200 OK 
{
  "carro": {
    "created_at": "2025-10-18T15:04:19.509111",
    "id": "cb7edbf7-763d-450b-bea0-9ed0eb7608a6",
    "license_plate": "FHH8346",
    "locale": "Subsolo 2, Vaga 12",
    "model": "Ford Ka",
    "parked": true
  },
  "mensagem": "Carro cadastrado com sucesso!"
}

Campos da Resposta

Campo Tipo Descrição
carro object Objeto contendo os dados do carro recém-cadastrado.
carro.id UUID ID único do registro de estacionamento.
carro.created_at string Data e hora da entrada (ISO 8601).
carro.license_plate string Placa do veículo.
carro.model string Modelo do veículo.
carro.locale string Localização da vaga.
carro.parked boolean Status de estacionado (true para entrada).
mensagem string Mensagem de confirmação.

Possíveis Erros

400 Bad Request 
{
  "mensagem": "Carro já cadastrado!"
}
Uso: Este endpoint registra a entrada de um novo veículo no estacionamento. O status parked será definido como true.

DELETE /vehicles/{plate} ▼
Registrar saída de um carro (Checkout)

Parâmetros de URL

Parâmetro Tipo Descrição Obrigatório
plate string Placa do veículo que está saindo. SIM

Headers

Nome Valor Obrigatório
Accept application/json SIM

Exemplo de Request (cURL)

curl --request DELETE \
  --url https://api.estacionar.com/vehicles/FHH8346 \
  --header 'Accept: application/json'

Resposta de Sucesso

200 OK 
{
  "data": {
    "created_at": "2025-10-18T15:04:20",
    "exit_hour": "Sat, 18 Oct 2025 15:07:17 GMT",
    "id": "cb7edbf7-763d-450b-bea0-9ed0eb7608a6",
    "license_plate": "FHH8346",
    "locale": "Subsolo 2, Vaga 12",
    "model": "Ford Ka",
    "parked": 1,
    "total_price": 7,
    "total_time": "2 minutos"
  },
  "message": "Veiculo excluído com sucesso."
}

Campos da Resposta

Campo Tipo Descrição
data object Objeto contendo os detalhes do "checkout" do veículo.
data.exit_hour string Data e hora da saída.
data.total_price number Valor total a ser pago.
data.total_time string Tempo total de permanência.
message string Mensagem de confirmação da saída.

Possíveis Erros

404 Not Found 
{
  "mensagem": "Carro não encontrado"
}
Importante: Este endpoint não apenas deleta o registro, ele executa a lógica de "checkout": calcula o tempo de permanência, o preço total, e marca o veículo como não estacionado (parked: false), retornando os detalhes da cobrança.

POST /price-parking ▼
Criar configuração de preços

Headers

Nome Valor Obrigatório
Accept application/json SIM
Content-Type application/json SIM

Parâmetros do Body (JSON)

Parâmetro Tipo Descrição Obrigatório
parking_hours integer Horas máximas de estacionamento (ex: 3). SIM
quick_stop_price number Valor da parada rápida (ex: 5). SIM

extra_hour_price number Valor da hora adicional (recorrência de 1 hora). SIM

curl --request POST \

  --header 'Content-Type: application/json' \
  --data '{
    "parking_hours": 3,
    "quick_stop_price": 5,
    "until_time_price": 13,
    "extra_hour_price": 4,
    "quick_stop_limit_minutes": 15
}'

Resposta de Sucesso

201 Created 
{
  "message": "Preço criado com sucesso!",
  "price_config": {
    "id": "uuid-price-config-123",
    "parking_hours": 3,
    "quick_stop_price": 5,
    "until_time_price": 13,
    "extra_hour_price": 4,
    "quick_stop_limit_minutes": 15

Códigos de Status HTTP

Código Significado Descrição
200 OK Requisição bem-sucedida
201 Created Recurso criado com sucesso
204 No Content Requisição bem-sucedida sem conteúdo de retorno
400 Bad Request Requisição inválida ou malformada
401 Unauthorized Credenciais inválidas ou token ausente
403 Forbidden Acesso negado ao recurso solicitado
404 Not Found Recurso não encontrado
406 Not Acceptable Erro de validação nos dados enviados
500 Internal Server Error Erro interno do servidor

Boas Práticas

  • Sempre inclua o header Accept: application/json em todas as requisições
  • Guarde o token de forma segura: Nunca exponha o token JWT em logs ou repositórios públicos
  • Valide os dados antes de enviar: Isso reduz erros de validação e melhora a performance
  • Use paginação adequada: Para listas grandes, utilize o header page e limit com valor apropriado
  • Trate os erros adequadamente: Sempre verifique o código de status HTTP e os detalhes de erro retornados
  • Implemente retry logic: Para falhas temporárias (500, timeouts), implemente tentativas automáticas
  • Monitore a expiração do token: O token JWT expira em 365 dias. Implemente renovação automática quando necessário
Suporte Técnico: Para dúvidas, problemas ou sugestões relacionadas à integração, entre em contato com a equipe técnica do Estacionar através dos canais de suporte fornecidos.
